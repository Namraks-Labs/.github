## Branch Protection Policy

All repositories in this org should follow these rules on their default branch (`main`):

- **No direct pushes** — all changes go through pull requests
- **1 required approval** before merging
- **Dismiss stale approvals** when new commits are pushed
- **Require approval of the most recent push**
- **Status checks must pass** and branches must be up to date before merging
- **Linear history required** — squash or rebase merges only
- **No force pushes** to protected branches
- **No branch deletion** of protected branches
- **Admin bypass allowed** — org admins can bypass when needed for solo projects

> **Note:** Branch protection is currently enforced on public repos. Private repos require GitHub Team plan ($4/user/month) for enforcement. Until upgraded, contributors to private repos should follow these rules by convention.
